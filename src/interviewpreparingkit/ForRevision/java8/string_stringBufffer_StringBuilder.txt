In Java, String, StringBuffer, and StringBuilder are classes used for working with text. They differ in terms of their immutability, performance, and usage. Here's a comparison of these classes along with examples:

1. String:

Immutability: Strings in Java are immutable, meaning that once a string object is created, its value cannot be changed. Any operation that appears to modify a string actually creates a new string object.
Usage: Strings are suitable for situations where the content won't change frequently. They are safe for use in multi-threaded environments because they are inherently thread-safe due to their immutability.
Example:
String str = "Hello, ";
str += "world"; // This creates a new string
System.out.println(str); // Output: "Hello, world"
======================================================================================================================================
2. StringBuffer:

Mutability: StringBuffer is mutable, meaning it can be modified without creating new objects. It is designed for situations where string content changes frequently and needs to be efficiently modified.
Usage: Use StringBuffer when you need to build or modify strings in a multi-threaded environment, as StringBuffer is synchronized.
Example:
StringBuffer buffer = new StringBuffer("Hello, ");
buffer.append("world");
System.out.println(buffer.toString()); // Output: "Hello, world"
======================================================================================================================================

3. StringBuilder:

Mutability: StringBuilder is similar to StringBuffer in terms of mutability but is not synchronized. This makes it more efficient in single-threaded applications but less suitable for multi-threaded scenarios.
Usage: StringBuilder is typically the preferred choice for string manipulation when thread safety is not a concern.
Example:
StringBuilder builder = new StringBuilder("Hello, ");
builder.append("world");
System.out.println(builder.toString()); // Output: "Hello, world"


Performance Considerations:

StringBuilder is usually the fastest among the three, as it is not synchronized and is designed for single-threaded performance.
StringBuffer is slightly slower than StringBuilder due to synchronization but is thread-safe.
String is the slowest for string concatenation and modification because it involves the creation of new objects for each modification.
In summary, choose the class that best fits your requirements:

Use String for immutable, unchanging text.
Use StringBuffer for mutable, multi-threaded text operations.
Use StringBuilder for mutable, single-threaded text operations with better performance.
