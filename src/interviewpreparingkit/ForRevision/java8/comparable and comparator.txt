In Java, Comparable and Comparator are interfaces that are used to define custom sorting orders for objects.
They are essential for sorting elements in collections like arrays, lists, and sets. Let's explore both of these interfaces with examples:

1. Comparable:
The Comparable interface is used to define a natural ordering for a class, meaning that the class itself specifies how its instances should be compared and sorted.
The natural ordering is typically based on a single attribute or criteria.

To make a class comparable, it should implement the Comparable interface and override the compareTo method, which takes another object of the same type as a parameter and returns a negative integer, zero, or a positive integer based on how the current object compares to the given object.

Here's an example using the Comparable interface:
import java.util.*;

class Person implements Comparable<Person> {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public int compareTo(Person otherPerson) {
        return this.age - otherPerson.age;
    }

    public String toString() {
        return name + " (Age: " + age + ")";
    }
}

public class ComparableExample {
    public static void main(String[] args) {
        List<Person> people = new ArrayList<>();
        people.add(new Person("Alice", 30));
        people.add(new Person("Bob", 25));
        people.add(new Person("Charlie", 35));

        Collections.sort(people);

        for (Person person : people) {
            System.out.println(person);
        }
    }
}


In this example, the Person class implements Comparable, and the compareTo method is used to compare instances based on their age. When we sort the people list using Collections.sort, it arranges the Person objects in ascending order of age.
------------------------------------------------------------------------------------------------------------------------------------------------------------------
2. Comparator:
The Comparator interface allows you to define custom comparison logic for classes that you don't control, or when you need multiple ways to sort objects. You can create multiple comparators for the same class.

To use a Comparator, you create a separate class that implements the Comparator interface and provides the comparison logic in the compare method.

Here's an example using a Comparator:

import java.util.*;

class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    public String toString() {
        return name + " (Age: " + age + ")";
    }
}

class AgeComparator implements Comparator<Person> {
    public int compare(Person person1, Person person2) {
        return person1.getAge() - person2.getAge();
    }
}

public class ComparatorExample {
    public static void main(String[] args) {
        List<Person> people = new ArrayList<>();
        people.add(new Person("Alice", 30));
        people.add(new Person("Bob", 25));
        people.add(new Person("Charlie", 35));

        Collections.sort(people, new AgeComparator());

        for (Person person : people) {
            System.out.println(person);
        }
    }
}


In this example, we've created an AgeComparator class that implements the Comparator interface and provides the logic to compare Person objects based on their age. We then use this comparator when sorting the people list.

Using Comparable and Comparator allows you to customize the sorting behavior of objects in Java collections according to your specific needs.
